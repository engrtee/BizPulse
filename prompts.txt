TASK: Add voice note and photo processing 
to BizPulse for opening stock setup and 
ongoing stock logging.

CONTEXT:
Nigerian traders are extremely comfortable 
with voice notes on WhatsApp. Asking them 
to type 20 to 30 product names during 
onboarding creates too much friction and 
causes drop-off before they ever log 
their first entry.

Voice notes and photos remove this barrier 
entirely. A trader speaks or snaps and 
BizPulse handles the rest.

This feature serves two purposes:
1. Opening stock onboarding — new user 
   declares their full stock in one 
   voice note or photo
2. Ongoing logging — any user can send 
   a voice note or photo to log 
   stock received or sales made

PART 1 — Opening stock onboarding flow

Every new user who registers at 
mybizpulse.app needs to log their 
opening stock before BizPulse can 
track anything meaningful for them.

Add a flag to the users table:
opening_stock_logged BOOLEAN DEFAULT false
opening_stock_logged_at TIMESTAMP

After the welcome message BizPulse 
sends this onboarding prompt:

"Before I can watch your stock for you 
I need to know what you currently have.

You can tell me three ways:

🎤 *Voice note* — just say it out loud:
'I have 20 bottles oud oil, 
15 rose oil, 5 musk oil...'

📸 *Photo* — snap your shelf or 
products and send it

✍️ *Type it* — 
'I have 20 oud oil 15 rose 5 musk'

Which is easiest? Send whenever ready. 📦"

Once opening stock is confirmed and logged:
Set opening_stock_logged = true
Set opening_stock_logged_at = NOW()

Only begin morning briefings after 
opening_stock_logged = true.

If user tries to log a sale before 
opening stock is logged send:

"Before I track your sales I need to 
know your starting stock.

Send me what you have right now — 
voice note, photo, or just type it.

Example: 'I have 20 oud oil 15 rose 5 musk'"

PART 2 — Voice note processing

Trigger: user sends an audio message 
(content type audio/ogg or audio/mpeg)

Processing flow:

Step 1 — Receive audio via WhatsApp webhook
Extract media_id from message object

Step 2 — Download audio from Meta:
GET https://graph.facebook.com/v18.0/
{media_id}
Headers: Authorization Bearer {token}
Returns: download URL

Download the audio file to temporary 
storage

Step 3 — Send to Gemini multimodal:

System prompt:
"You are processing a voice note from 
a Nigerian small business owner. 
They may speak in English, Nigerian 
Pidgin, Yoruba-inflected English, 
Igbo-inflected English, or Hausa-
inflected English.

They are either:
A) Declaring their current stock 
   (what they have right now)
B) Logging stock they just received 
   from a supplier
C) Logging sales they just made

Listen carefully and extract every 
product mentioned with:
- Product name (normalise to standard 
  English where possible)
- Quantity (number + unit)
- Price per unit if mentioned
- Transaction type: 
  opening_stock / stock_in / sale

Nigerian product names to watch for:
Oud oil, rose oil, musk oil, 
ankara fabric, lace fabric,
Indomie noodles, Peak Milk, 
Cowbell milk, Golden Morn,
Caprisonne, Cabin biscuits,
garri, rice, beans, palm oil,
tomatoes, pepper, onions

Return ONLY valid JSON array.
No explanation. No markdown.
No text before or after the JSON.

Format:
[{
  product: string,
  quantity: number or null,
  unit: string or null,
  price: number or null,
  type: opening_stock|stock_in|sale
}]

If you cannot extract any clear 
product information return: []"

Step 4 — Parse JSON response
If array is empty: send cannot 
understand message (see Part 5)

If array has items: continue to 
confirmation step

Step 5 — Apply Nigerian product 
name normalisation (same dictionary 
as product fuzzy matching in 
Option C prompt)

Step 6 — Send confirmation message:

"Here is what I heard 🎤

[For opening_stock type:]
📦 *Your current stock:*
- [Product]: [quantity] [unit]
- [Product]: [quantity] [unit]
[repeat for all products]

Reply *YES* to set this as your 
starting stock ✅
Reply *EDIT* if something is wrong ❌
Or send a correction"

[For stock_in type:]
"📦 *Stock received:*
- [Product]: [quantity] [unit] 
  at ₦[price] each
  Total cost: ₦[amount]
[repeat]

Reply *YES* to log ✅
Reply *EDIT* if wrong ❌"

[For sale type:]
"💰 *Sales logged:*
- [Product]: [quantity] at ₦[price]
  Revenue: ₦[amount]
[repeat]

Reply *YES* to log ✅
Reply *EDIT* if wrong ❌"

Step 7 — On YES:
If opening_stock: set current_stock 
for each product in products table
Set opening_stock_logged = true

If stock_in or sale: log as 
product_transaction (same flow 
as text-based entries)

PART 3 — Photo processing

Trigger: user sends an image
Content type: image/jpeg or image/png

Determine intent from caption:
Caption contains "stock"/"received"/
"delivery"/"new stock"/"I get" 
→ treat as stock receipt

Caption contains "my products"/
"what I have"/"my stock" or 
no caption + new user 
→ treat as opening stock declaration

Caption contains "sold"/"sales" 
→ treat as sale entry

No caption + existing user 
→ treat as opening stock update 
  and ask to confirm

Processing flow:

Step 1 — Receive image via webhook
Extract media_id

Step 2 — Download from Meta media endpoint

Step 3 — Send to Gemini Vision:

System prompt:

Estimate quantities only if clearly 
countable. Mark uncertain items.

For receipts and invoices:
Extract all line items including 
product name, quantity, unit, price.
Handle handwritten text carefully.
Nigerian currency is naira (₦ or N).

For handwritten stock lists:
Extract all product names and 
quantities listed.

Return ONLY valid JSON array.
No explanation. No markdown.

Format:
[{
  product: string,
  quantity: number or null,
  unit: string or null,
  price: number or null,
  confidence: high|medium|low,
  notes: string or null
}]

Mark confidence as:
high — clearly visible and readable
medium — visible but partially unclear
low — guessed or unclear

If image is too dark, blurry, or 
contains no product information: 
return []"

Step 4 — Parse JSON response

Step 5 — Apply product normalisation

Step 6 — Send confirmation with 
confidence indicators:

"Here is what I can see in your photo 📸

✅ Clear:
- [Product]: [quantity] [unit]
  at ₦[price]
[repeat for high confidence items]

❓ Not sure about these:
- [Product]?: [quantity] [unit]
[repeat for low confidence items]

Reply *YES* to log everything ✅
Reply *EDIT* to make changes ❌
Or tell me what I got wrong"

Step 7 — On YES:
Log all items regardless of confidence.
Log was confirmed by user so 
responsibility is theirs.
